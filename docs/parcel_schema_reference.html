Every table that joins to a parcel

Skagit County publishes parcel data as several independently-loaded feeds — the assessor export, the auditor's recorded-document index, the county's GIS parcel layer, WAZA zoning, and levy/tax rolls. This is the full set, organized the way you'd actually pull one parcel's story: record → land & improvements → sales & recordings → geometry & zoning → tax history.

20tables
~6.0Mtotal rows
2join-key families
83,646parcels on record

Two join-key families, one identifier

parcel_number

Assessor-derived tables key on parcel_number — or a lowercase parcelnumber on land/improvements, inherited from the source export's column naming. e.g. P74512

parcel_id

GIS-derived tables (loaded from the county's shapefiles, not the assessor export) key on parcel_id instead. Same value, same format — verified: 77,304 of 83,646 assessor rows match a gis_skagit_parcels.parcel_id row directly. e.g. P74512

Coverage gap: not every parcel_number has a matching parcel_id geometry — about 5,900 parcels (7%) have no GIS shape on file, which is exactly what parcel_geo_static_features.feature_status = 'missing_coordinates' tracks.
